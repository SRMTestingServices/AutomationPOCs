## 🚀 How It Works

The repository includes a GitHub Action (`.github/workflows/generate-index.yml`) that:

1. **Triggers** automatically when:
   - Any `.html` file is pushed to `main` or `master` branch
   - Can also be triggered manually via "Actions" tab

2. **Scans** the repository root for all `.html` files

3. **Generates** a beautiful `index.html` with:
   - Sorted list of all HTML files
   - Direct links to each file
   - Timestamp of last update
   - Responsive design

4. **Commits** the updated `index.html` back to the repository

## 📝 Usage

### Adding New HTML Files

Simply add your HTML files to the repository root and push:

```bash
git add your-new-file.html
git commit -m "Add new HTML file"
git push
```

The GitHub Action will automatically:
- Detect the new file
- Regenerate `index.html`
- Commit and push the updated index

### Manual Trigger

You can also manually trigger the index generation:

1. Go to the **Actions** tab in your GitHub repository
2. Select **Generate Index HTML** workflow
3. Click **Run workflow**

## 🎨 Viewing the Index
